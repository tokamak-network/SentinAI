LLM-Enhanced Playbook Evolution Guide

SentinAI의 AI 기반 플레이북 자동 생성 및 진화 기능을 대시보드에서 확인하고 활용하는 방법

Overview

SentinAI는 운영 이력에서 패턴을 학습하고, LLM을 활용해 최적화된 remediation 플레이북을 자동 생성합니다.

Heuristic (기존)

  • 규칙 기반 confidence 계산
  • 성공률/빈도 기반 플레이북 생성
  • 매일 00:05 UTC 자동 실행

LLM Enhanced (신규)

  • AI가 패턴 분석 후 최적 구성 생성
  • 액션 순서, 타임아웃, fallback 최적화
  • Provider fallback (Qwen → Anthropic → OpenAI → Gemini)

3. Playbook Lifecycle

Review Workflow
DRAFT
AI 생성 직후
→
APPROVE
APPROVED
검증 완료
→

자동 실행 가능
Status설명자동 실행버튼
DRAFT LLM 또는 패턴 마이닝으로 생성됨, 검토 필요 No APPROVE / SKIP
APPROVED 운영자가 승인함, 프로덕션 사용 준비 Manual only MARK TRUSTED / SUSPEND
TRUSTED 완전 신뢰, 조건 충족 시 자동 실행 Yes SUSPEND
SUSPENDED 일시 정지됨 No —
TRUSTED 상태의 플레이북만 이상 징후 발생 시 자동으로 실행됩니다. AI가 생성한 플레이북은 항상 DRAFT로 시작하므로, 운영자 검토 없이 실행되지 않습니다.

4. API로 직접 확인

API Endpoints

Evolution 상태 조회

# 플레이북 현황 (대시보드와 동일 데이터)
curl https://sentinai.tokamak.network/thanos-sepolia/api/playbook-evolution?action=status

# 응답 예시:
{
  "ledger": { "total": 847, "lastTimestamp": "2026-03-17T06:25:45Z" },
  "playbooks": { "total": 5, "needsReview": 2 },
  "scheduler": { "nextCron": "00:05 UTC (daily)" }
}

수동 패턴 마이닝 + LLM 생성 트리거

# 패턴 마이닝 + LLM 플레이북 생성
curl -X POST https://sentinai.tokamak.network/thanos-sepolia/api/playbook-evolution?action=mine \
  -H "x-api-key: $API_KEY"

# 응답 예시:
{
  "patterns": 3,
  "saved": ["pb-cpu-spike", "pb-gas-ratio"],
  "skipped": [],
  "llmEvolved": {
    "generatedBy": "anthropic/claude-sonnet-4-5-20250929",
    "name": "Optimized CPU Spike Response",
    "actions": 2,
    "fallbacks": 1
  }
}

Admin: LLM Evolution 트리거 + 버전 승격

# LLM 진화 트리거 (패턴 분석 → LLM 생성 → 자동 버전 승격)
curl -X POST https://sentinai.tokamak.network/thanos-sepolia/api/admin/playbook-evolution \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"action": "trigger_evolution"}'

# 응답 예시:
{
  "action": "evolution_triggered",
  "patterns": [
    { "anomalyType": "high_cpu", "successRate": 92, "executionCount": 50 }
  ],
  "evolved": {
    "promoted": true,
    "versionId": "v-3",
    "generatedBy": "anthropic/claude-sonnet-4-5-20250929"
  }
}

# 특정 버전으로 롤백

6. Architecture Summary

LLM Evolution Pipeline
PatternMiner → analyzeRecords() → PlaybookEvolver → chatCompletion() → RollbackManager → promoteVersion()
ComponentRoleAI?
PatternMiner 운영 이력에서 반복 패턴 추출 (anomalyType + action 그룹핑) No
PlaybookEvolver 패턴을 기반으로 LLM이 최적화된 플레이북 JSON 생성 Yes
RollbackManager 버전 관리, 승격, 롤백 (최대 10 버전 보관) No
ABTestController 새 버전 vs 기존 버전 A/B 테스트 (Fisher's exact test) No
