## v2 대비 변경 사항

### 전략 전환: SaaS 구독 → Agent-for-Hire

v2는 전통적인 SaaS 모델을 계획했다 (General 무료 / Premium $299/체인/월 / Enterprise).
평가 결과, 이 모델의 근본적 약점이 드러났다: **셀프호스팅 오픈소스 소프트웨어는 기능 제한을 강제할 수 없다.**

v3는 **Agent-for-Hire** 모델을 채택한다:
- 소프트웨어는 무료 오픈소스 (변경 없음)
- 수익은 검증 가능한 실적을 가진 **숙련된 에이전트 고용**에서 발생
- 경험 기반 가격: Trainee (무료) → Junior ($199) → Senior ($499) → Expert ($799)
- 성과 기반 추가 과금: 검증된 결과(자동 해결 인시던트, SLA 달성)에 대해서만 과금

### v2에서 삭제된 항목

| v2 항목 | 상태 | 사유 |
|---------|------|------|
| 구독 티어 (General/Premium/Enterprise) | **삭제** | 경험 기반 에이전트 가격으로 대체 |
| Feature Gate (명예 시스템) | **삭제** | 제한할 기능 없음 — 모든 코드가 공개 |
| 결제 스텁 (StubPaymentProvider) | **삭제** | 성과 기반 청구 이벤트로 대체 |
| 트라이얼 넛지 시스템 (60/80/85/90일) | **삭제** | 에이전트 가격이 투명하므로 트라이얼 불필요 |
| 가격 페이지 (/pricing) | **재설계** | 구독 티어가 아닌 에이전트 등급 표시 |

## 1주차 (Day 1–7): 에이전트 지능 + 수익 인프라

### Day 1–2: Experience Transfer Protocol

> 신규 에이전트가 동일 프로토콜 타입의 패턴을 50% 신뢰도로 시작한다.
> 이것이 "숙련된 에이전트를 고용하라"의 핵심 가치 제안이다.

```
파일:
  생성: src/lib/experience-transfer.ts
  생성: src/lib/__tests__/experience-transfer.test.ts
  수정: src/types/experience.ts (TransferablePattern 타입 추가)
  수정: src/types/redis.ts (IStateStore 메서드 추가)
  수정: src/lib/redis-store.ts (전이 저장소 구현)

핵심 로직:
  extractTransferablePatterns(protocolId: string): Promise<TransferablePattern[]>
    → protocolId로 experience store 쿼리
    → 신뢰도 > 0.7이고 발생 횟수 > 5인 패턴 추출
    → 인스턴스별 데이터 제거 (IP, URL, 운영자 정보)
    → 익명화된 패턴 스키마 + 성공률 반환

  bootstrapNewAgent(instanceId: string, protocolId: string): Promise<BootstrapResult>
    → 해당 프로토콜 타입의 전이 가능 패턴 가져오기
    → 50% 신뢰도 할인 적용하여 주입
    → experience store에 전이 이벤트 기록
    → 반환: { patternsTransferred, sourceProtocol, discountApplied }

전이 흐름:
  Agent(Thanos, 6개월) 47개 패턴 보유
    → 신규 OP Stack 체인 온보딩
    → bootstrapNewAgent()로 47개 패턴을 50% 신뢰도로 전이
    → 신규 체인에서 패턴 검증 시 신뢰도 → 100%
```

```
파일:

  생성: src/lib/__tests__/collective-intelligence.test.ts

    → 모든 인스턴스의 패턴을 프로토콜 타입별로 병합

    → 집단 라이브러리에서 일치하는 패턴 검색
    → 신뢰도 × 성공률로 순위 매기기

프라이버시 모델:
  공유:     패턴 스키마 + 성공률 + 프로토콜 타입 + 발생 횟수
  비공유:   원시 메트릭, RPC URL, 운영자 신원, 재무 데이터
  모든 데이터는 로컬 유지 (셀프호스팅) — 집계는 인스턴스별 컬렉션 단위
```

### 오픈소스가 도움이 되는 이유 (해가 아님)

1. **코드 투명성** → 고객이 에이전트가 무엇을 하는지 검증 → 신뢰 → 전환
2. **락인 없음** → 낮은 도입 장벽 → 더 많은 운영자가 trainee 등급 시도
3. **커뮤니티 기여** → 더 많은 체인 플러그인 → 더 큰 대상 시장
4. **해자는 데이터, 코드가 아님** → 6개월의 운영 경험은 `git clone`할 수 없다
