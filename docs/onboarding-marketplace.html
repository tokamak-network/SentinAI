E2E Flow
Transaction Structure
5 parties, 3 phases — from discovery to on-chain TON settlement.
① Discovery — one-time setup
② Per-call Payment — every request
③ Batch Settlement — periodic
① When does discovery run?
Once, when a buyer first encounters this operator. After caching agent.json, the buyer knows all 7 services and their prices.
② When does payment happen?
The EIP-712 signature is verified instantly off-chain on the backend. Data is returned immediately. No waiting for a block.
③ When does TON transfer?
The Facilitator contract batches authorizations and executes on-chain TON transfers periodically. Operator receives TON in bulk.
Section 2
ERC8004 Registry
How operators register their identity on-chain.
Smart Contract (3 functions)
WRITE register(agentURI)
Called once by the operator. Stores their service manifest URL and emits an AgentRegistered event with the new agentId.
READ latestAgentIdOf(addr)
Look up the most recent agentId for a wallet address. Returns 0 if not registered.
READ agentUriOf(agentId)
Resolve agentId → agentURI. Points to /api/agent-marketplace/agent.json
What gets registered

How the server resolves the operator address
Priority 1
🔑 MARKETPLACE_WALLET_KEY
Private key env var. Address auto-derived. Also used for on-chain tx.
Priority 2
📋 OPERATOR_ADDRESS
Explicit public address. No private key needed. For hardware wallets.
Priority 3
💾 Redis (cached)
Saved after wallet-connect registration. Key: marketplace:registry:operator-address
⚠️
Registration status check requires two env vars Even if the address resolves correctly, the server returns registered: false unless both ERC8004_REGISTRY_ADDRESS and L1_RPC_URL are set — it can't query the blockchain without them.

Connect Wallet
MetaMask → eth_requestAccounts. Validates Sepolia (chainId: 11155111).
2
Fetch 402
GET endpoint without payment → parse amount, token, facilitator contract address.
3
Approve TON
Check TON balance + allowance. If allowance too low → ERC-20 approve() tx.
4
Sign EIP-712
eth_signTypedData_v4 in MetaMask. Generates the authorization signature.
5
Retry + Receive
GET with X-PAYMENT header → 200 + data. Modal shows JSON response.
🔐 Admin Login (SIWE)
1
Connect Wallet
Click [CONNECT WALLET] on /login. MetaMask opens.
2
Fetch Nonce
GET /api/auth/siwe/nonce?address=0x... → 32-char hex, 5-min TTL.
3
Sign SIWE Message
EIP-4361 message: domain, address, nonce, timestamp. User approves in MetaMask.
4
POST /verify
Server checks: nonce fresh, signature valid, address = MARKETPLACE_WALLET_KEY.

Cookie Issued
sentinai_admin_session (HttpOnly, 8hr). Redirected to /v2/marketplace.

Section 7
Setup & Environment Variables
The minimum set to get a working marketplace.
✅ Quickstart Checklist
☑ ERC8004_REGISTRY_ADDRESS
☑ L1_RPC_URL (Sepolia)
☑ OPERATOR_ADDRESS or MARKETPLACE_WALLET_KEY
☑ NEXT_PUBLIC_OPERATOR_API_URL (website)
ERC8004_REGISTRY_ADDRESS REQUIRED Contract address on Sepolia. Default: 0x64c8f8cB66657349190c7AF783f8E0254dCF1467
L1_RPC_URL REQUIRED Ethereum Sepolia RPC (Alchemy / Infura). Needed for all on-chain reads.
MARKETPLACE_WALLET_KEY OPTIONAL Operator private key (0x...). Enables server-side ERC8004 registration. Address derived automatically.
OPERATOR_ADDRESS OPTIONAL Explicit public address. Use when registered from hardware wallet (no private key on server).
MARKETPLACE_AGENT_URI_BASE OPTIONAL Base URL for agentUri submitted to ERC8004. Example: https://sentinai.tokamak.network/thanos-sepolia
MARKETPLACE_PAYMENT_MODE OPTIONAL Payment verification mode. facilitated (default, EIP-712) · stub (testing) · open (no auth — ⚠️ never in prod)
NEXT_PUBLIC_OPERATOR_API_URL REQUIRED (website) Backend URL used by the Vercel website. Must include basePath, e.g. https://sentinai.tokamak.network/thanos-sepolia
🖥️ Backend (Docker)
  • sentinai.tokamak.network
  • Node 20 Alpine, 3-stage build
  • Port: 3002 internally
  • Healthcheck: GET /api/health
  • Requires: Redis for persistence
🌐 Website (Vercel)
  • sentinai-xi.vercel.app
  • Root dir: website/
  • Standard Next.js on Vercel
  • No Docker, no Redis needed
  • Must set NEXT_PUBLIC_OPERATOR_API_URL
