# SentinAI Tier Comparison: General vs Premium

> 현재 구현 기준 | 작성일: 2026-02-23

---

## 요금 설계

### 과금 단위: 보호 중인 체인(Chain) 수

SentinAI의 가치 단위는 "보호하는 L2 체인 수"입니다. 사용자 수나 API 호출량이 아닙니다.

```
고객이 느끼는 가치 = "내 체인이 안 죽었다"
                  = 다운타임 방지 가치
                  = 절약된 엔지니어링 시간
```

### 기본 요금

| | **General** | **Premium** | **Enterprise** |
|--|:-----------:|:-----------:|:--------------:|
| **월 요금** | 무료 | $299 / 체인 / 월 | 별도 협의 |
| **보호 체인 수** | 1개 | 체인당 과금 | 무제한 |
| **지원** | 커뮤니티 | 이메일 (48h SLA) | 전담 담당자 + SLA |

> **$299 근거**: 시니어 DevOps 1인 월급 $8,000 ÷ 27일 ≈ 하루 $296. 체인 1개를 수동 운영하는 하루 비용이 SentinAI 한 달 보호 비용과 같습니다.

### 특별 정책

#### Rollup Hub 파트너십

Hub에서 appchain을 배포한 팀은 SentinAI와 자동 연동되며, 첫 **90일 무료**로 Premium을 사용합니다. 90일 이후 전환 제안을 받습니다.

```
Hub appchain 배포 → SentinAI 자동 연동 → 90일 무료 → Premium 전환 제안
```

무료 기간은 비용이 아닌 고객 확보 비용으로 취급합니다. Hub 배포 팀이 늘어날수록 SentinAI 잠재 고객이 자동으로 증가합니다.

#### TON 토큰 결제 혜택

| 결제 수단 | 혜택 |
|---------|------|
| TON 토큰 월간 결제 | 15% 추가 할인 |
| TON 토큰 연간 선결제 | 25% 추가 할인 |

### 가격 포지셔닝

| 서비스 | 대상 | 월 요금 |
|--------|------|:-------:|
| Datadog (인프라 모니터링) | 일반 서버 | ~$15/host |

L2 체인의 다운타임 비용(기회비용 + 신뢰 손실)은 일반 서버보다 훨씬 크기 때문에 $299는 보수적인 가격입니다.

### 업셀링 경로

```
General (무료, 1체인)
  │  "새벽 장애를 다음날 아침에 알았다"
  ▼

  │  "체인이 3개가 됐는데 대시보드를 여러 개 열어야 한다" (Q3 이후)
  ▼
Premium 볼륨 할인 (멀티체인)
  │  "전담 SLA가 필요하다 / 커스텀 Playbook이 필요하다"
  ▼
Enterprise (별도 협의)
```

## 핵심 철학

| | 일반 (General) | 프리미엄 (Premium) |
|--|--------------|-----------------|
| **슬로건** | "SentinAI가 24/7 대신 지켜드립니다" | "SentinAI가 24/7 대신 운영해드립니다" |
| **의사결정 주체** | AI가 감지, 사람이 판단 | AI가 감지·판단·실행까지 |
| **AI 깊이** | 통계 기반 | 분석 + 예측 + 자율 목표 수립 |
| **데이터 보존** | 인메모리 (재시작 시 초기화) | Redis 영구 저장 |

> **운영 자동화는 양 티어 모두 제공합니다.** 일반은 기본 자율화(Agent Loop), 프리미엄은 AI 강화 + 전문 에이전트입니다.

## 시나리오별 경험 차이

| 상황 | 일반 사용자 경험 | 프리미엄 사용자 경험 |
|------|----------------|------------------|
| 새벽 TxPool 급증 | 스케일링은 됐지만 원인 불명 | AI가 원인 분석 후 결과 보고 |
| 장애 직전 조짐 | 임계치 초과 시 알림, 사후 대응 | 예측 모델이 사전 감지 → 선제 스케일업 |
| Batcher EOA 잔액 부족 | 알림 수신 후 수동 충전 | 임계치 전 자동 충전 |
| L1 RPC 429 에러 | 장애 발생 후 수동 endpoint 전환 | 감지 즉시 자동 전환, L2 중단 없음 |
| 원인 불명 성능 저하 | 운영자가 로그 직접 분석 | RCA 엔진이 의존성 경로 자동 추적 |
| 새벽 3시 이상 발생 | 다음날 아침 발견 | Agent Loop 즉시 감지 + 자동 복구 + 보고 |

## 자연스러운 업그레이드 유도 지점

무료 사용자가 한계를 느끼는 순간이 곧 업셀링 기회입니다.

| 한계 경험 | 프리미엄 메시지 |
|----------|--------------|
| "스케일링은 됐는데 왜 발생했는지 모르겠다" | → AI RCA가 원인까지 찾아드립니다 |
| "장애가 나고 나서야 알았다" | → 예측 스케일링이 사전에 막아드립니다 |
| "새벽에 수동으로 EOA 충전했다" | → 자동 충전으로 운영자 개입이 없습니다 |
| "RPC 죽을 때마다 직접 바꾼다" | → 자동 Failover로 L2가 멈추지 않습니다 |
| "매일 아침 로그 보는 게 일과다" | → AI 일간 리포트로 30초 확인으로 끝납니다 |
